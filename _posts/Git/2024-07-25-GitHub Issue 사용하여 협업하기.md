# 0. 이슈 관리가 중요한 이유?

> 새로운 기능 요청, 버그 보고, 개선 사항 등을 이슈로 생성할 수 있다.

- 문제 추적 및 관리
  - 버그, 기능 요청, 개선 사항 등 다양한 문제를 문서화하고 정리할 수 있어 프로젝트 진행 상황을 명확하게 파악할 수 있다.<br>
- 협업 효율성
  - 누구나 현재 어떤 문제가 있는지, 누가 무엇을 하고 있는지 쉽게 확인할 수 있다.<br>
- 문서화
  - 이슈를 통해 프로젝트의 진행 과정, 결정을 내린 이유, 해결된 문제 등을 문서화할 수 있다.

<br><br>

# 2. 템플릿 예시

## 2.1 버그 리포트 이슈 템플릿(Bug Report Issue Template)

- Template name: 🐞Bug
- About: Bug 발생 시 작성해 주세요.

```markdown
# 어떤 버그인가요?

> 어떤 버그인지 간결하게 설명해주세요

## 어떤 상황에서 발생한 버그인가요?

> (가능하면) Given-When-Then 형식으로 서술해주세요

## 예상 결과

> 예상했던 정상적인 결과가 어떤 것이었는지 설명해주세요

## 📍레퍼런스

- [Title](https://...)
```

<br>

## 2.2 기능 추가 이슈 템플릿(Feature Add Issue Template)

- Template name: ✅Feature request
- About: Feature 작업 상황을 입력해주세요.

```markdown
## 📄이슈 내용

- 이슈 내용 요약 설명

## 📝상세 내용

- 이슈 내용 구현 관련 상세 내용 작성

## ✅체크리스트

- [ ] TODO A
- [ ] TODO B
- [ ] TODO C

## 📍레퍼런스

- [Title](https://...)
```

<br>

## 2.3 질문 이슈 템플릿(Question Issue Template)

```markdown
# 1. 질문 제목

> 질문의 핵심을 간결하게 설명하는 제목을 작성해주세요.

## 2. 질문 내용

> 질문을 상세히 설명해주세요. 문제를 겪고 있는 상황, 구체적인 질문, 또는 해결하고자 하는 점을 포함시켜 주세요.

## 3. 현재 상태

> 질문과 관련된 현재 상황을 설명해주세요.
>
> - **문제 발생 위치**: 문제가 발생하는 위치나 부분
> - **현재 시도한 해결 방법**: 지금까지 시도한 방법이나 접근 방식
> - **에러 메시지**: 관련된 에러 메시지나 경고 (있다면)

## 4. 기대 결과

> 질문을 해결했을 때 기대하는 결과나 바라는 상태를 설명해주세요.

## 5. 참고 자료 (선택)

> 질문에 도움이 될 수 있는 자료나 링크를 제공해주세요.
>
> - 예: 관련 문서, 코드 스니펫, 스크린샷 등

## 6. 기타

> 추가적으로 질문과 관련된 정보나 특이사항이 있다면 적어주세요.
```

<br>

## 2.4 작업 요청 이슈 템플릿(Task Request Issue Template)

```markdown
# 1. 작업 제목

> 요청하는 작업의 간결한 제목을 작성해주세요.

## 2. 작업 목적

> 이 작업이 필요한 이유와 목표를 설명해주세요.

## 3. 작업 상세 내용

> 작업의 우선 순위를 포함해주세요 (예: 긴급, 높음, 중간, 낮음).

## 4. 필요한 자료

> 작업에 참고할 자료나 추가적인 정보가 있다면 제공해주세요.
>
> - 예: 문서 링크, 디자인 파일, 예제 코드 등

## 5. 예상 일정

> 작업을 완료할 예상 일정을 작성해주세요.
>
> - **시작일**: 작업을 시작할 날짜
> - **완료일**: 작업을 완료해야 하는 날짜

## 6. 담당자

> 작업을 담당할 사람이나 팀의 정보를 작성해주세요.

## 7. 기타

> 추가적으로 필요한 정보나 특이사항이 있다면 적어주세요.
```

# 4. Github 이슈와 커밋 메시지 연결하기

## 4.1 이슈 번호 확인

이슈 생성 후 생성된 이슈 번호를 확인하자. 이슈 제목 뒤에 붙은 번호가 (ex. `#1` )이 이슈 번호이다.

![](/assets/images/2024/2024-07-25-23-27-31.png)

<br>

## 4.2 이슈에 커밋 반영하기

① 브랜치 생성 및 이동

```
git checkout feat/1   // (처리기능/이슈번호)
```

② 스테이지에 파일 추가

```
git add .
```

③ 커밋하기<br>
: 커밋 메시지를 `[커밋 타입/#이슈번호] - 커밋 내용` 형태로 작성하자

```
git commit -m "[feat/#1] - issue test"
```

④ push하기

```
git push -u origin feat/1
```
